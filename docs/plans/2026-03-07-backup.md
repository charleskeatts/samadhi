# Backup & Disaster Recovery Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Auto-backup Clairio code to Dropbox (zip snapshot) and GitHub (git push) on every login and daily at 9am — zero manual effort after setup.

**Architecture:** A shell script does the work (zip to Dropbox + git push); a macOS LaunchAgent schedules it. Two files total, activated with two Terminal commands.

**Tech Stack:** Bash, macOS LaunchAgent (plist), zip, git

---

### Task 1: Create the backup script

**Files:**
- Create: `~/scripts/clairio-backup.sh`

**Step 1: Create the scripts directory**

```bash
mkdir -p ~/scripts
```

Expected: no output, directory created.

**Step 2: Write the script**

Create `~/scripts/clairio-backup.sh` with this exact content:

```bash
#!/bin/bash
set -euo pipefail

PROJECT="/Users/charleskeatts/Projects/samadhi"
DROPBOX="$HOME/Library/CloudStorage/Dropbox"
BACKUP_DIR="$DROPBOX/Clairio Backups"
DATE=$(date +%Y-%m-%d)
ZIPFILE="$BACKUP_DIR/clairio-$DATE.zip"
LOG="$HOME/Library/Logs/clairio-backup.log"

echo "$(date): Starting Clairio backup" >> "$LOG"

# Create backup dir if it doesn't exist yet
mkdir -p "$BACKUP_DIR"

# Zip the project — skip node_modules, build output, and secrets
zip -r "$ZIPFILE" "$PROJECT" \
  --exclude "*/node_modules/*" \
  --exclude "*/.next/*" \
  --exclude "*/.env.local" \
  --exclude "*/.env.production.local" \
  --exclude "*/.env.development.local" \
  2>> "$LOG" \
  && echo "$(date): Zip saved → $ZIPFILE" >> "$LOG" \
  || echo "$(date): WARNING — zip failed" >> "$LOG"

# Delete zips older than 14 days (keeps storage lean)
find "$BACKUP_DIR" -name "clairio-*.zip" -mtime +14 -delete 2>> "$LOG" \
  && echo "$(date): Old backups cleaned up" >> "$LOG"

# Push any unpushed git commits to GitHub (non-fatal if auth not yet set up)
cd "$PROJECT"
git push origin main >> "$LOG" 2>&1 \
  && echo "$(date): Git push succeeded" >> "$LOG" \
  || echo "$(date): Git push skipped (non-fatal — check GitHub auth)" >> "$LOG"

echo "$(date): Backup run complete" >> "$LOG"
```

**Step 3: Make it executable**

```bash
chmod +x ~/scripts/clairio-backup.sh
```

Expected: no output.

**Step 4: Run it manually to verify**

```bash
~/scripts/clairio-backup.sh
```

Expected: completes in ~30 seconds with no errors printed to screen.

**Step 5: Check the log**

```bash
cat ~/Library/Logs/clairio-backup.log
```

Expected output (dates will vary):
```
Sat Mar  7 09:00:00 PST 2026: Starting Clairio backup
Sat Mar  7 09:00:28 PST 2026: Zip saved → /Users/charleskeatts/Library/CloudStorage/Dropbox/Clairio Backups/clairio-2026-03-07.zip
Sat Mar  7 09:00:28 PST 2026: Old backups cleaned up
Sat Mar  7 09:00:29 PST 2026: Git push succeeded  ← or "skipped" if GitHub auth not done yet
Sat Mar  7 09:00:29 PST 2026: Backup run complete
```

**Step 6: Verify the zip landed in Dropbox**

```bash
ls ~/Library/CloudStorage/Dropbox/Clairio\ Backups/
```

Expected: `clairio-2026-03-07.zip` (or today's date)

### Task 2: Create the LaunchAgent (scheduler)

**Files:**
- Create: `~/Library/LaunchAgents/com.clairio.backup.plist`

**Step 1: Write the plist**

Create `~/Library/LaunchAgents/com.clairio.backup.plist` with this exact content:

```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
    <key>Label</key>
    <string>com.clairio.backup</string>
    <key>ProgramArguments</key>
    <array>
        <string>/bin/bash</string>
        <string>/Users/charleskeatts/scripts/clairio-backup.sh</string>
    </array>
    <key>RunAtLoad</key>
    <true/>
    <key>StartCalendarInterval</key>
    <dict>
        <key>Hour</key>
        <integer>9</integer>
        <key>Minute</key>
        <integer>0</integer>
    </dict>
    <key>StandardOutPath</key>
    <string>/Users/charleskeatts/Library/Logs/clairio-backup.log</string>
    <key>StandardErrorPath</key>
    <string>/Users/charleskeatts/Library/Logs/clairio-backup.log</string>
</dict>
</plist>
```

**Step 2: Load the LaunchAgent**

```bash
launchctl load ~/Library/LaunchAgents/com.clairio.backup.plist
```

Expected: no output (silence = success on macOS).

**Step 3: Verify it's registered**

```bash
launchctl list | grep clairio
```

Expected output:
```
-    0    com.clairio.backup
```

The `-` means it's loaded (not currently running). The `0` means last exit code was 0 (success). If you see a non-zero exit code, check the log.

## If you ever need to disable the backup

```bash
launchctl unload ~/Library/LaunchAgents/com.clairio.backup.plist
```

## If you ever need to re-enable it

```bash
launchctl load ~/Library/LaunchAgents/com.clairio.backup.plist
```

## Recovery playbook

| What broke | Recovery |
|---|---|
| Deleted a file | `unzip ~/Library/CloudStorage/Dropbox/Clairio\ Backups/clairio-YYYY-MM-DD.zip` |
| Laptop died | Download zip from dropbox.com → unzip → `npm install` |
| Need an old commit | `github.com/charleskeatts/samadhi` → commits → browse/download |
